# Privacy Policy

trip-json is an open-source tool that runs locally on your machine. It does not collect, transmit, or store any personal data, telemetry, or analytics.

## What this project does

- Runs locally as a Claude Code plugin or as a cloned repository
- Generates trip JSON files by calling the Google Places API with **your** API key (governed by Google's policies)
- Curates places using [Claude Code](https://claude.ai/claude-code), which is subject to Anthropic's privacy policy
- Renders maps using Mapbox with **your** API token
- Outputs HTML files that you choose where (and whether) to share

## What this project does NOT do

- No telemetry, analytics, crash reporting, or usage tracking
- No phoning home — no server, no backend, no accounts
- Does not store, transmit, or share your API keys anywhere except your local `.env.local` file (gitignored)
- Does not store, transmit, or share your trip data anywhere — trip JSONs and built HTML files live on your disk and are uploaded only to hosts you explicitly choose

## Third-party services

When you use this project you interact with the following services, each governed by their own privacy policies:

- **[Google Maps Platform](https://policies.google.com/privacy)** — Geocoding API, Places API. Calls happen during trip creation only. Your queries (place names) are sent to Google.
- **[Mapbox](https://www.mapbox.com/legal/privacy)** — vector tiles, fonts, sprites. Calls happen every time the renderer loads in a browser. Mapbox sees the requesting IP and the token used.
- **[Anthropic Claude](https://www.anthropic.com/legal/privacy)** — only when you invoke `/research-trip` or `/static-trip` via Claude Code. Anthropic processes the prompt and tool calls.

You are responsible for complying with each service's terms.

## Trip data and shared files

Trip JSONs are plain text files on your disk. Self-contained HTML files generated by `/static-trip` include your Mapbox token, the trip data, and the renderer code. If you share these files, the recipient's map loads use **your** Mapbox quota — please see the README's rate-limit guidance.
